**Learning Outcomes**

---

## Project Overview

This project focused on developing a **University Shift Management System** as part of the *Interface Pessoa-Máquina* course at Universidade do Minho. The system aims to streamline the management of course schedules and shift allocations for students and course directors, minimizing conflicts and enhancing usability through intuitive interfaces.

Key functionalities include:  

- User authentication with registration and login for students  
- Shift visualization and swap requests for students  
- Comprehensive dashboard for course directors to manage schedules, notifications, and manual allocations  
- Automated and manual shift allocation with conflict detection  
- Notification system for shift change requests and system updates  

### Main Features:
- Role-based interfaces for students (profile, schedule view) and course directors (dashboard, notifications, manual allocation)  
- Responsive design prototyped in Figma for accessibility and usability  
- Support for manual and automated shift scheduling with conflict resolution  
- Clear feedback mechanisms for user actions (e.g., confirmation messages, error alerts)  
- Adherence to usability heuristics for intuitive navigation and error prevention  

---

## Implementation

- **Tools:** Figma for prototyping, HTML/CSS/JavaScript (assumed for implementation based on Figma designs)  
- **Interface Design:** 
  - Student interfaces: Registration, Login, Profile, and Schedule pages  
  - Course Director interfaces: Dashboard, Notifications, Manual Allocation, Schedules, Students, and Shifts pages  
- **Navigation:** Sidebar menu for quick access to sections, with a logout button for session management  
- **Functionalities:** 
  - Students can view schedules and request shift swaps via pop-ups  
  - Course directors manage shift allocations, resolve conflicts, and handle notifications  
- **Usability:** Followed Nielsen’s usability heuristics, including visibility of system status, error prevention, and user control  

---

## Testing & Results

- Usability testing to validate intuitive navigation and feedback clarity  
- Functional testing of registration, login, and shift management processes  
- Conflict detection tested for manual allocations, ensuring error messages for full shifts or scheduling conflicts  
- User feedback confirmed ease of use for both students and course directors  
- Metrics included successful shift allocation rates and user satisfaction with interface responsiveness
